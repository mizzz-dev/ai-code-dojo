# 作業ログ運用（正本）

最終更新: 2026-05-14

## このディレクトリの目的
`docs/logs/` は、Issue/PR 単位の作業事実を時系列で記録し、会話履歴に依存せずに意思決定と実装経緯を追跡できる状態を維持するための正本ディレクトリです。

## 保管対象
- Issue ごとの着手メモ（背景、スコープ、非目的）
- 実装中に発生した判断ログ（代替案、採用理由、未確定事項）
- テスト実行結果の要約（成功/失敗と原因）
- handoff に必要な引き継ぎ事項（次アクション、ブロッカー）
- AI 利用時の補助ログ（どの prompt で何を生成・検証したかの要点）

> 詳細な prompt 本文は `docs/ai-prompts/` を正本とし、ここでは参照リンクと要約のみを扱う。

## ファイル命名規則
- 形式: `YYYY-MM-DD-issue-<番号>-<短い要約>.md`
- 例: `2026-05-14-issue-31-docs-logs-bootstrap.md`
- 要約は小文字ケバブケースを推奨し、`a-z0-9-` の範囲で命名する。

## 更新タイミング
1. **着手時**: 対象 Issue の目的・非目的・開始時点の前提を記録する。
2. **主要判断時**: 設計や運用判断を行ったタイミングで追記する。
3. **PR 作成時**: 実施内容・テスト結果・未対応事項を確定する。
4. **handoff 時**: 次担当が即時再開できる粒度で残課題を明記する。

## Issue / PR / AI prompt / handoff との関係
- Issue: ログ先頭に `Issue #番号` を明記し、課題トレーサビリティを確保する。
- PR: 対応 PR 番号と差分概要を記載し、レビュー導線を残す。
- AI prompt: `docs/ai-prompts/` の該当ファイルをリンクし、重複保存を避ける。
- handoff: 次アクション・判断保留点・確認観点をチェックリストで残す。

## 正本ドキュメント導線
- 全体方針と Canonical Source Rules: `docs/project-overview.md`
- 現況スナップショット: `docs/current-status.md`
- 進行中課題と優先順位: `docs/active-issues.md`
- AI prompt 履歴: `docs/ai-prompts/`
- ADR（設計判断）: `docs/adr/`

## 運用ルール
- 既存ログは履歴として保持し、追記を基本とする（必要時のみ訂正履歴を明記）。
- 個人情報・secret・hidden tests の実データは記載しない。
- 仕様の正本は各 Canonical Source に委譲し、ログ側へ仕様本文を重複記載しない。
